前言

刚刚磨完ES6的我正打算学习Vue时却发现JS已经过时了，现在的前端的逻辑语法（尤其是Vue）已经被一种更加完美的语言所代替了——也就是最新的Typescript，Typescript本质上是JavaScript的超集，在实际使用中也是将TS转化为JS来使用，不过最重要的区别在于对于某些错误（尤其是类型错误）JS在编译时并不会报错，而在运行时才报错，这对Debug或者说团队协作开发来说非常不友好。而TS遇到这种错误时会在编译阶段直接指出，即时的反应出错误在哪里（在团队协作中不至于去祸害其他人，哈哈），对代码的规范化起到了很好的作用，因此TS正在被越来越多的大型企业或项目使用。

故友说：学习是不能停下来的，对此我表示认同。

关于原型

d表示为派生类，b表示基类。

即 class d extends b{…}。

我们需要了解以下关键：

  1. _proto_
  2. prototype
  3. new在函数调用时对其内部的this的影响。
  4. new对1、2的影响

JS的所有对象均有__proto__这一成员，通俗的来讲，__proto__是一条连接父类的绳索，在查找某个属性（例如x.age）时，如果在x上找不到这个属性，那么它就会顺着__proto__去它的父类寻找，即x.__proto__.age，如果还找不到，就继续在往上寻找:x.__proto__.__proto__.age，直到age被找到，或者age为null时，才会停止寻找。

而JavaScript所有函数都有一个prototype属性，prototype上有个指向函数本身构造器属性。

简单来说，定义在函数内部的函数会在每次被实例化时都会被初始化，举例来说，现在有以下函数：

这与JS的“立即调用表达式”有关，指类或者有些函数会在js运行时第一时间执行，即在这个时候初始化。

箭头函数

=>函数，也叫做胖函数，基本特性是：()=>{…}，它的特点如下：

  1. 不再需要书写function
  2. 包含了this的含义
  3. 包含了argument的含义

箭头函数的意义并不只是少写function而已，而是在函数中明确了this真正的意义，之前的函数里的this会指向函数的拥有者或执行者，这导致在使用一些特殊的函数（如子函数）时this经常指向window，从而找不到数据，以前的解决方法是在函数外将“还正确的”this赋给另一个量（如that）来使用，很是麻烦。

而箭头函数的出现取缔了这种类似“脱裤子放屁”的步骤，箭头函数将会根据上下文this的意义来确定this的意义。

但本质上是和之前的解决方法是一样的，不过箭头函数自己处理了。

我们一般建议使用“===”，原因大家也清楚，在JS里使用“==”时有时会发生类型的隐式转换，比如：

因为JS在比较两个类型不同的数据时，会将其类型转为一致再进行比较，例如这里字符串的“5”就会被转为数字5，最终返还true。而TS则不会发生这种隐式转换。

TS基础类型

数据类型 关键字 描述
任意类型 any 声明为 any 的变量可以赋予任意类型的值。
数字类型 number 双精度 64 位浮点值。它可以用来表示整数和分数。
字符串类型 string 一个字符系列，使用单引号（'）或双引号（"）来表示字符串类型。反引号（`）来定义多行文本和内嵌表达式。
布尔类型 boolean 表示逻辑值：true 和 false。
数组类型 无 声明变量为数组。// 在元素类型后面加上[] let arr: number[] = [1, 2]; // 或者使用数组泛型 let arr: Array<number> = [1, 2];
元组 无 元组类型用来表示已知元素数量和类型的数组，各元素的类型不必相同，对应位置的类型需要相同。let x: [string, number]; x = ['Runoob', 1]; // 运行正常 x = [1, 'Runoob']; // 报错 console.log(x[0]); // 输出 Runoob
枚举 enum 枚举类型用于定义数值集合。enum Color {Red, Green, Blue}; let c: Color = Color.Blue; console.log(c); // 输出 2
void void 用于标识方法返回值的类型，表示该方法没有返回值。function hello(): void { alert("Hello Runoob"); }
null null 表示对象值缺失。
undefined undefined 用于初始化变量为一个未定义的值
never never never 是其它类型（包括 null 和 undefined）的子类型，代表从不会出现的值。

**注意：**TypeScript 和 JavaScript 没有整数类型。

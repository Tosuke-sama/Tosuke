首先让我们明白React的渲染过程：

  1. 利用babel将jsx转化为js
  2. 利用React对象的createElement方法来创建虚拟DOM，需要三个参数：
    type(标签类型)、props(属性)、children(子节点)
  3. 转换为真实DOM
  4. 利用render方法渲染

manifest.json 指定了开始页面 index.html，“提供有关应用程序的信息，如名称、作者、图标或图标、描述。

index.html是开始页面，一切的开始都从这里开始，所以这个是代码执行的源头。

打开APP.js，这里就类似于Vue里的主页面App. vue，我们能在index.js中看见App的引用：

App.js是一个页面组件，和Vue一样是页面套页面、组件套组件，但在React中整合他们的不是一个另一个大的页面，而是一个index.js的引导文件，我们在index.js里导入并使用app.js，对app.js的使用也是标签使用，也就是<App />这个标签。

参照图中App标签的位置，具体如何使用是依靠的ReactDom对象的渲染方法，也就是ReactDOM.render(…) ，所有的 js,html 都可通过它进行渲染绘制，他有两个参数，内容和渲染目标 js 对象。

内容就是要在渲染目标中显示的东西，可以是一个 React 部件，也可以是一段HTML或TEXT文本。渲染目标JS对象，就是一个DIV或TABEL,或TD 等HTML的节点对象。

图片中这种创建模式是在创建ReactDom时就已经给定了JSDom了，所以render方法也是直接加入需要渲染的标签就好。

unmountComponentAtNode() 这个方法是解除渲染挂载，作用和 render 刚好相反，也就清空一个渲染目标中的 React 部件或 html 内容。

类式的封装有些不同：

  1. 它必须拥有自己的render方法，用于输出组件
  2. 类式的封装第一个字母必须大写

React18中，如果不遵循这个书写规范，实际上并不会发生什么

但是在一些旧版本中，会出现报错

所以还是按着规范来，而且良好的书写规范在实际合作开发中是有利于代码阅读理解的

3.类式的封装只能包含一个顶层标签，这与vue的template相似，即最外层只能有一个标签。

4.类式的封装的属性可以在this.props上获取

5.class需要写作className，lable标签中for需要写作htmlFor，因为class和for是JS的保留字。
